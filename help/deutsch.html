Help Center

1 - Inventory Section

a) General informations

In the inventory part you can list and consult the hardware and software inventory park.

You are able to perform parameterized and sorted researches on one of the types of the inventory (computers, networks, printers, monitors, cartridges, peripherals and softwares).

Here, you can post the form containing precise information of each indexed item, or add an item.

You have then the possibility to modify or delete each item.

Inside each kind of forms (computers, printers etc.), tabs are available to sorts the informations to be more clear and readable. You can chose the "ALL" tab to see all the availables informations.

The tabs << & >> let you browse between the differents items of your inventory (Using the ID as condition).

b) Templates Management

You can define models of each type of items in your park, this allowing you to add several similar items very easily.

c) Management of the computers internals devices

Since GLPI 0.5, the notion of internal devices has appear. This let you add as much devices as you need in a computer.

You can manage the list of all the internal devices of your park in the Setup/Internal devices section.

Some internal devices type let you specify some specifics informations such as the hardware capacity.

d) Management of the direct connections

The direct connections make it possible to model the relation between the central processing unit of a computer and external components.

A direct connection is done between a computer and a printer, a monitor or an external device.

The connections or disconnections are done since the form of a computer or that of the dependent material.

e) Management of the networking ports

A port network makes it possible to model a network interface on a given material.

Each port has an IP address, a mac address and a nework point to which it is connected.

The network connections are done by connecting two ports. To connect two materials together they must have an available port.

In most of the configurations, the connections will be done between a networking port on a computer, a screen or a printer and a networking port on a network material (hub, switch)

When connecting two materials, informations such as the IP address and the network point are updated for both of the ports

For each material, it is possible to add one or several ports

f) Management of the softwares

A software can have several licences having each one an associated serial and an end date

The installation of a software is done since the formsheet of a computer.

Each time that a software is installed, the number of licence available for this software decreases.

It is impossible to install a software, when no licence is available.

Nevertheless, it is possible to indicate the keyword "free" as serial when you add a new licence to a software. This license is installable without worrying about the number of the licenses.

its also possible to use the keyword "Global" when you are using a global licence your entire park.

It is possible to add several licences in one time. The serial and the end date will be the same one for each license added.

There are many options available when creating a software, you can specify if the software is OEM, bought or if its an upgrade to another software.

g) Management of the material states

In this section you can see the states of all the material. This section can be used to add or remove and item from the proposal states.

Also to put a state to a material, you must go into the formsheet of this material and use the dropdown "states" and chose the appropriate state then click the update button to confirm it. This will place the item into the "Material states" section.

To remove the state of a material, simply do the same thing as when you add a state to a material but simply chose the "---" and click the update button.

2 - Assistance Section

a) Tracking

The tracking module makes it possible to post the list of the interventions according to several criteria.

You can show:

  • All active interventions

    • All the interventions which were not closed.

  • Only intervention assigned to you

    • The interventions which were assigned to you.

  • Only interventions not assigned yet

    • The interventions which must be assigned to a technician.

  • Only old interventions

    • The closed interventions.

You can also perform a research on the interventions based on a description or the ID number.

If interventions are found, GLPI shows a table including the interventions. It shows for each intervention: its status (new or old),the ID number, the opening and close date, the priority level, the author of the request, the person to which was assigned the intervention, the affected material, the category,the description of the request and the informations available on the followup of the intervention.

A click on one of these elements will enable you to reach additional information.

To add elements of followup to an intervention, to close it or to reassign it, click on "infos". Then, You will obtain the complete history of the intervention. You will be able to add an element of followup or close the intervention (optional: indicate the real time of intervention).

Particular case of the closed interventions

The restoration of a finished intervention is possible. Click on "information" then "Restore".

It is possible to remove closed interventions if you do not wish to preserve them in the database. For that, check the checkbox in the status column and click on "Delete" at the bottom of the list of the interventions.

b) Helpdesk

The helpdesk is use to add the interventions requests. You can use this section with all type of glpi users.

Its possible to state the lvl of priotity of the intervention, the ID number of the software or hardware in trouble and the type of intervention.

If you activate the emails followup in the GLPI settings, another field will be available to enter the email of the request person.

You can see the interventions in the Tracking section of GLPI.

3 - Management Section

a) Contacts

You can enter all your contacts in this section and classify them between the commercial or technicien type. Secondly you can associate those contacts with the companies you are doing business.

b) Enterprises

In this section, its possible to classify all the companies and their third party type who have some relations with your company and associate the contacts of those companies. To add or modify the third party dropdown, go into the GLPI setup section under the dropdown option.

c) Contracts

Within this section, you can manage all types of contracts who are associated with the material of your park and also the informations related to these contracts. Here are the possibles types of contract : loan, location, leasing, insurance, hardware and sofware maintenance and the prestation.

You can associate the created contracts to all elements in your inventory from the contract section or the formsheet of an item.

d) Documents

This section let you save some importants documents to your GLPI server then you can associate these documents to all materials or software in your inventory and classify them between differents headings. It is possible to specify the MIME type of the document if this one is not present already.

To add or modify the headings list, go into the Administration and setup section under the dropdown option, chose the "item state".

4 - Utils Section

a) Rerservations

This section will show you what material is reservable.

You can add some comments to the reservables materials. To do it, click on the "modify comment" button.

To set materials to be reservable, simply go the formsheet of the material and click on the "Authorize the loan" link.

For the Helpdesk users, a section is available to see the reservables materials.

b) Knowledge database

The knowledge database has two goals. The first one is the centralize all the knowledge of your business to all techniciens. The second is to give users informations to be able to solve their problems themself.

To be able to dissociate those objectives, the notion of public FAQ is available. From this, only the elements into the public FAQ is available to all users. The elements not defined in the public FAQ are only available from the central.

You can create your own category and sub-category for a better consultation later on. Its then easy for someone to retreive the information needed for his work.

c) Reports

GLPI authorizes the generation of a certain number of reports:

  • The default report

    • This report lists the number of item and software present in the inventory.

  • Material under maintenance contract

    • This report indicates the number of item under maintenance contract. It is possible to obtain a detailed report by type of item and date of purchase. The multiple selection is authorized.

  • Report by purchase date or end of guarantee

    • This report indicates the number of item by purchase date or end of guarantee for one or several years. The multiple selection is authorized.

  • Report on interventions

    • This report makes it possible to obtain the list of the interventions and their history according to many criteria's. Research can be perform over a given period, for a precise type of item or for a part of this item and according to its author and/or of his assign technician.

  • Network report

    • This report show you the list of informations concerning your network by the location, the switch or the network point.

  • Excel, OpenOffice (Sylk)

    • GLPI can generate exploitable reports by type of into unsing a spreadsheet. For that GLPI generates SYLK files. This format is exploitable by many spreadsheves.

    If you used a software which does not know well with the standards (Excel for example), use the limited version in order not to have a problem with the size of the fields. The long fields will be truncated.

5 - Administration Section

a) Users Management

This part works exactly in the same way that the items of the inventory. You can add, modify and remove users.

b) Setup

    i) Dropdowns

    You can in this section completly modify the data included in the select lists for example for the locations and the processors.

    Once selected the list that you want to modify you can add of the elements, modify them and remove them.
    Caution !! If you remove elements used, then, you will have to make a choice between modifying the existing entries or to put them at NULL.

    ii) Internals perepherals

    Here you can modify all the internals devices that you can add into computers.

    iii) External authentifications

    The external authentifications enable you to use alternative sources of authentification. That avoids you having to add all the users manually.

    The first solution is to use a mail server:
    - In the first part, you must to define the mail server you want to use. Please, to fill it correctly follow this link php.net which provide several examples. Some exwmples : {localhost:110/pop3}, {localhost:993/imap/ssl}, {localhost:995/pop3/ssl/novalidate-cert}...
    - The second parameter defines the mail domain used which will be used automatically to fill the mail addresses of the new users. If the user is called toto, then, his mail address will be toto@mail_domain.

    The second solution id to use an LDAP server or an Active Directory. For this configuration defer to corresponding documentation present on the Web site of GLPI.

    iv) Email Followup

    This section enables you to activate the followup of the interventions by email. Data (Save/Restore) If you activate this functionality, emails will be sent for the creation of a new intervention, the addition of each followup and the closing of the interventions.
    - You should fill the email address of the administrator and the email signature which will be added to each email send
    - The other options enable you to limit the emails sent. For each type of people you can activate or not the sending for the creation of an intervention, the addition of followups and the closing of an intervention.
    - The defined groups of people are: the administrator, all 'admin' or 'normal' users, the person in charge of the task and the caller of the intervention.
    - The caller will not receive an email if it do not activated the email followup when he request for the intervention.

    v) General Configuration

    The general configuration of the application makes it possible to define certain parameters:
    - The installation path. Caution !! This path is detected automatically at the time of the installation or of the update, you should not need to modify it.
    - The log level.
    - The number of events to be posted corresponds to the number of lines of log present on the banner page of the central.
    - The time of conservation of the logs. Caution!! If you keep them a long time your table can contain lots of data.
    - Show the interventions at the login: to or not to see the active interventions on the banner page of the central.
    - Number of elements ti print by page (valid for all the lists of elements posted - Tracking, computers...)
    - Maximum number of characters for each elements of the list: reduce the size of the character strings to show in order to obtain a readable posting.
    - Allow anonymous tracking post (helpdesk.html): activate or not the anonymous requests for interventions via the helpdesk.html file.

    vi) Document Type

    Within this section, you can add or modify the documents types authorized to be managed by GLPI. The needed informations are :
    - The name of the document type.
    - The extention ex: .txt
    - the name of the icon file of the document. Place the icons files in this folder pics/icones under the GLPI path.
    - The MIME type if needed.
    - Authorization of download of this file type (yes or no).

c) Settings

This section allows you to change your password, to define the default order used in the posting of the interventions and to define the language used.

d) Data (Backup/Restore)

This zone allows you to create SQL safeguards of the database ("Dump SQL").

You can also remove the backup files, to restore them or to download SQL files.

You can also generate a XML safeguard of the database. ("Dump XML").
